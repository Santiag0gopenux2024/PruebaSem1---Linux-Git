DIFERENCIAS ENTRE GIT REBASE Y MERGE.

GIT MERGE.
Funciona de una manera en la que unificamos los cambios de todos los stach con master, pero manteniendo el historial de versiones de la rama.






GIT REBASE.
Funciona de una manera en la que unificamos los cambios pero superponiendo el branch sobre master y al mismo tiempo eliminando las versiones de la rama, ya que todas haran parte de master
